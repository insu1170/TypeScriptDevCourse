## HTTP 메서드

- 생성(Create): **POST**
- 조회(Read): **GET**
- 수정(Update): **PUT**, **PATCH**
- 삭제(Delete): **DELETE**

> PUT vs PATCH
> 
- PUT: 자원 전체 수정 (덮어쓰기 개념)
- PATCH: 자원 일부 수정

---

## Node.js 특징

- **싱글 스레드**
    - 하나의 메인 이벤트 루프에서 동작
    - 요청을 순서대로 처리하는 것이 아니라 **이벤트 큐** 기반
- **이벤트 기반(Event-driven)**
    - 특정 이벤트(요청, 응답, 파일 읽기 등)가 발생했을 때 콜백 실행
- **논블로킹 I/O**
    - I/O 작업이 끝날 때까지 기다리지 않음
    - 오래 걸리는 작업은 백그라운드로 넘기고 다음 코드 실행

> 장점: 동시 요청 처리에 매우 강함
> 
> 
> 단점: CPU 연산이 많은 작업에는 부적합
>

## 모듈(Module)

- 특정 기능을 수행하는 **코드 묶음**
- 재사용 가능

### 라이브러리 vs 프레임워크

- **라이브러리**: 필요한 기능만 가져다 씀
- **프레임워크**: 정해진 구조 안에서 코드를 작성 (제어권이 프레임워크에 있음)

---

## 콜백 함수 (Callback Function)

- 함수의 매개변수로 **함수를 전달**
- 나중에 특정 시점에 실행됨

예시 개념:

> "이 작업 끝나면 이 함수 실행해줘"
>

## NPM (Node Package Manager)

- Node.js 패키지(모듈) 관리 도구
- 외부 모듈 설치, 삭제, 버전 관리

### 주요 파일

- **package.json**: 프로젝트 정보 및 의존성 목록
- **package-lock.json**: 실제 설치된 정확한 버전과 다운로드 경로 기록

> npm은 registry URL에서 패키지를 다운로드 → 압축 해제 → node_modules에 설치
> 

---

## NPM 명령어

- `npm install 모듈명` : 로컬 설치
- `npm install -g 모듈명` : 글로벌 설치
- `npm uninstall 모듈명` : 모듈 삭제

> 글로벌 설치: 모든 프로젝트에서 사용 가능 (CLI 도구 위주)
>
