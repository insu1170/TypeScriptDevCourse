## forEach (향상된 for문)

- 배열을 순회하며 각 요소에 대해 **콜백 함수 실행**
- 반환값(`return`) 없음

```jsx
const a = [1, 2, 3];

// data: 현재 요소
// index: 인덱스
// all: 원본 배열
a.forEach((data, index, all) => {
  console.log(data, index);
});

```

- 반복 목적(출력, 상태 변경 등)에 적합

---

## map

- 배열을 순회하면서 **새로운 배열을 생성**
- `return` 값이 새로운 배열의 요소가 됨

```jsx
const a = [1, 2, 3];
const b = a.map((data) => data * 2);

console.log(b); // [2, 4, 6]

```

---

## forEach vs map 차이

| 구분 | forEach | map |
| --- | --- | --- |
| 반환값 | 없음 | 새로운 배열 |
| 목적 | 반복 실행 | 데이터 변환 |
| 체이닝 | 불가 | 가능 |

## 리팩토링 (Refactoring)

- **기능은 그대로 유지**하면서 코드 내부 구조를 개선하는 작업

### 리팩토링 목적

- 코드 가독성 향상
- 성능 개선
- 안정성 및 유지보수성 향상

### 리팩토링 시점

- 에러가 반복적으로 발견될 때
- 기능 추가 전
- 코드 리뷰 시
- 리팩토링 중 다른 문제를 발견할 수 있음

## HTTP 상태 코드

### 2xx (성공)

- `200 OK`
    - 조회 성공
    - 수정 / 삭제 성공
- `201 Created`
    - 리소스 생성(등록) 성공

---

### 4xx (클라이언트 오류)

- `404 Not Found`
    - 요청한 리소스를 찾을 수 없음

---

### 5xx (서버 오류)

- `500 Internal Server Error`
    - 서버 내부 오류
    - 서버가 정상적으로 요청을 처리하지 못한 경우
