## 표현식 (Expression)

- 어떤 값으로 평가되는 코드
- 값으로 이행되는 유효한 코드
- 예: 리터럴 표현식, 함수 표현식

## 문 (Statement)

- 프로그램을 구성하는 최소 실행 단위
- 예: 할당문, 선언문, 반복문

## 블록문 (Block Statement)

- 여러 명령문을 그룹으로 묶은 것
- `{ }` 로 구성

## throw 문

- 예외를 발생시키기 위해 사용
- throw 이후 코드는 실행되지 않음

## try / catch / finally

- try : 실행 시도
- catch : 오류 발생 시 실행
- finally : 마지막에 항상 실행

## 객체 (Object)

- 속성을 가진 독립적인 개체
- 속성들의 집합

### 생성자 함수

- `new` 키워드로 객체 생성
- 객체 인스턴스를 생성
- `this` 는 생성될 인스턴스를 가리킴

### 속성 조작 (동적 생성)

- 존재하지 않는 속성에 값을 할당하면 자동으로 생성

### 객체 순회

- `for...in` : 모든 열거 가능한 속성 순회
- `Object.keys(obj)` : 객체 자신의 열거 가능한 속성 이름을 배열로 반환
- `Object.getOwnPropertyNames(obj)` : 객체 자신의 모든 속성 반환

## 객체 복사

### 같은 객체 할당

- 변수에 그대로 할당하면 같은 주소를 참조

### 얕은 복사 (Shallow Copy)

- 복사된 객체가 내부 값 일부를 같은 참조로 공유

### 깊은 복사 (Deep Copy)

- 모든 속성이 새로운 값으로 복사

# Class

## Class

- 기존 프로토타입 기반 상속 메커니즘을 추상화한 문법
- 프로토타입 기반 패턴의 "문법적 설탕(Syntactic Sugar)"
- class 내부에는 메서드만 정의 가능
- 인스턴스는 `new` 키워드로 생성

## constructor

- 인스턴스 생성 후 클래스 필드를 초기화하는 특수 메서드
- 인스턴스 생성 시 자동 호출
- 전달된 인자를 constructor 매개변수로 받음

## 인스턴스 메서드

- 클래스 내부에 정의된 메서드
- `this` 를 통해 인스턴스에 접근

## 인스턴스 속성

- 클래스 내부에 캡슐화된 변수 (멤버 변수)
- `this` 에 바인딩하여 사용
- 보통 `constructor` 에서 초기화

## 상속 (extends)

- 코드 재사용성을 위해 사용
- 부모 클래스를 상속받는 클래스 정의

## super

- 부모 클래스의 생성자를 호출
- 부모의 인스턴스 속성을 초기화
- 부모와 공통된 변수 초기화 시 사용

# this 와 화살표 함수

## this

- 현재 실행 컨텍스트의 객체를 참조하는 키워드
- 어떤 객체인지 호출 방식에 따라 결정

## 함수 컨텍스트

- 함수 호출 시 this가 함께 전달

### 함수 호출 방식

- 일반 함수 호출
- 객체 메서드 호출
- 생성자 함수 호출
- 메서드가 아닌 일반 함수로 호출되면
- `this` 는 전역 객체를 가리킬 수 있음
- `bind()` 를 사용하면 this를 강제로 바인딩 가능

![image.png](attachment:4dad1eef-bc42-4876-ac3d-f7eafc70dd47:image.png)

- getAge는 매서드가 아닌 함수로 인식하기 때문에 this가 전역을 가르키게 된다
- 바인드를 통해서 this를 강제한 것

## 스코프

- 변수가 유효한 범위

## 스코프 규칙

### 렉시컬 스코프

- 변수가 선언된 위치에 따라 스코프 결정
- JavaScript 기본 규칙

### 다이나믹 스코프

- 호출 위치에 따라 스코프 결정 (JS는 사용하지 않음)

## 스코프 종류

- 전역 스코프
- 모듈 스코프
- 파일 스코프
- 함수 스코프
- 블록 스코프 (`let`, `const`)
