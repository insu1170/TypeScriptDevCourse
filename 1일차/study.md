## 프로젝트

프로젝트란 어떤 프로그램이나 서비스를 만들기 위한 전체 과정.

(기획 → 설계 → 구현 → 테스트 → 배포 → …)

- 프로젝트의 크기는 **범위** 를 어떻게 설정하느냐에 따라 달라진다.
- 기능 추가, 성능 개선, 리팩토링 같은 고도화 작업도 프로젝트에 포함.

---

## 협업의 중요성

프로젝트에서 가장 중요한 요소 중 하나는 **협업**이다.

협업 ⇒ 소통 ⇒ 공유

- 코드 공유
- 문서 공유
- 이슈 및 진행 상황 공유

이를 위해 등장한 것이 **README 파일**.

---

## README & Markdown(MD)

### README란?

- 프로젝트를 처음 접하는 사람이 한눈에 이해할 수 있도록 만든 **프로젝트 설명서**
- 개요, 실행 방법, 사용 방법, 진행 현황 등을 기록

### Markdown(MD)

- 웹에서 사용되는 HTML 태그보다 **쉽고 간단하게 문서를 작성**할 수 있도록 만든 것.
- 제목, 목록, 코드 블록 등을 간단한 문법으로 표현할 수 있어 협업 문서에 적합하다.

- 실습
    
    ```tsx
    **굵게**
    **_기울 굵_**
    
    # 1단계 제목
    ## 2단게 제목
    ###  3단계 제목
    ...
    ---
    ```
    // 프로그래밍 코드는 백틱 3번
    ```
    
    > 인용
    ```
    

**굵게**

***기울 굵***

# 1단계 제목

## 2단게 제목

### 3단계 제목

## ...


## 버전 관리

### 버전이란?

- **유의미한 변경 사항이 기록된 상태**를 의미한다.
- 보통 기능 추가, 수정, 개선 단위로 버전이 만들어진다.

### 버전 관리의 중요성

- 파일 손상이나 실수 발생 시 **이전 상태로 복구(백업)** 가능
- 변경 이력을 추적하여 협업 시 충돌을 줄일 수 있다.

---

## 버전 관리 시스템 (Version Control System, VCS)

### 로컬 VCS

- 개인 컴퓨터에서만 버전을 관리
- 로컬 환경에 한정되어 있어 **협업X**

### 중앙집중식 VCS

- 하나의 서버에서 모든 버전을 관리
- 사용자는 서버에서 파일을 내려받아 수정 후 다시 업로드
    - 파일 별로 관리하다 보니 버전 충돌 등과 같은 일이 발생할 수 있음

### 분산 VCS (Git)

- 중앙 저장소가 존재하지만, 각 사용자가 **프로젝트 전체 이력을 포함한 저장소를 로컬에 복제**한다.
