- 인터프리터 언어

### 동적 타입 언어

변수의 타입이 값에 따라 런타임에 결정됨

## 일급 객체 (First-class Object)

JavaScript에서 함수는 일급 객체이다.

즉 함수는 다음과 같이 사용 가능하다.

- 변수에 할당 가능
- 객체의 프로퍼티로 사용 가능
- 배열의 요소로 사용 가능
- 함수 호출의 인수로 전달 가능
- 함수의 반환값으로 사용 가능

## 프로토타입 기반 상속

JavaScript는 클래스 기반이 아닌 프로토타입 체인을 통해 객체 간 상속을 구현한다.

## 프로그래밍 패러다임

JavaScript는 여러 패러다임을 지원한다.

- 명령형 프로그래밍
- 함수형 프로그래밍
- 객체지향 프로그래밍

## 변수

데이터를 저장하는 메모리 주소를 식별하기 위한 이름

### var 특징

- 키워드 생략 가능
- 중복 선언 가능
- 변수 호이스팅 발생

### 변수 호이스팅

변수 선언이 코드의 최상단으로 끌어올려진 것처럼 동작하는 현상

문제점

- 전역 변수화 가능
- 예측하기 어려운 스코프 문제

### let / const

- 블록 스코프를 가짐
- 중복 선언 불가

실행 단계

1. 코드 평가 단계 : 변수 선언
2. 코드 실행 단계 : 초기화 및 값 할당

## 데이터 타입

### 원시 타입 (Primitive Type)

- number
- string
- boolean
- null
- undefined
- symbol

특징

- 값 변경 불가
- 값 자체가 전달됨

### 객체 타입 (Reference Type)

- object
- array
- function

특징

- 참조 방식으로 전달
- 속성 변경 가능

JavaScript에서 **원시 타입을 제외한 모든 것은 객체**이다.

## 타입 변환

JavaScript는 동적 타입 언어이므로 타입 변환이 발생한다.

### 명시적 타입 변환

개발자가 의도적으로 변환

### 암묵적 타입 변환

JavaScript 엔진이 자동으로 변환

예시

- `+ ""` → 문자열 변환
- `1` → 숫자 변환
- `!!` → 불리언 변환

## 연산자

하나 이상의 표현식을 대상으로 연산을 수행하여 값을 생성하는 것

종류

- 단항 연산자
- 산술 연산자
- 관계 연산자
- 이진 논리 연산자
- 삼항 연산자
- 할당 연산자
- 옵셔널 체이닝 연산자
- 쉼표 연산자

### 단항 연산자

하나의 피연산자만 사용하는 연산자

예시

- `typeof` : 값의 타입을 문자열로 반환
- `delete` : 객체의 속성 삭제
- `void` : 표현식이 값을 반환하지 않도록 함

## 함수

특정 작업을 수행하는 코드 블록

외부 코드에서 호출 가능한 하위 프로그램

### 함수 특징

JavaScript에서 함수는 일급 객체이다.

따라서

- 함수의 인수로 전달 가능
- 함수의 반환값 가능
- 변수에 할당 가능
- 비교 연산의 대상 가능

### 매개변수와 인자

- 매개변수(parameter) : 함수 정의 시 사용하는 변수
- 인자(argument) : 함수 호출 시 전달하는 실제 값

### 기본값 매개변수

매개변수에 기본값을 설정할 수 있다.

```jsx
function sum(a = 0, b = 0) {
  return a + b
}
```

### 나머지 매개변수

가변 인자를 받을 때 사용

```jsx
function sum(...nums) {
  return nums.reduce((a,b) => a + b, 0)
}
```

### arguments 객체

함수 호출 시 전달된 모든 인자를 담고 있는 유사 배열 객체

- 과제
    
    ```jsx
    const a = (...b) =>{ // 스프레드 연산자
    console.log(b)
    }
    
    a(1,2,3,4) // 1,2,3,4
    ```
