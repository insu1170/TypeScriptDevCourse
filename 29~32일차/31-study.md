# 31일차 Express & DB

## 📌 Express 라우팅

- 같은 경로에 여러 핸들러를 등록하면 **앞에 있는 핸들러부터 실행**된다.
- `next()`를 호출하지 않으면 다음 핸들러는 실행되지 않는다.

 같은 라우트를 사용해야 할 경우:

- 내부에서 `if` 문이나 조건 분기를 통해 서로 다른 기능을 수행하도록 만들 수 있다.

---

## 📌 DB 컬럼명 주의

- SQL 예약어(예: `order`, `group`, `select`)는 컬럼명으로 사용하지 않는 것이 좋다.

### 이유

- 쿼리 작성 시 오류 발생 가능
- 백틱(``)을 사용해야 하는 번거로움
- 가독성 저하

<aside>

라우팅은 실행 흐름을 이해하고 설계하고, DB 컬럼명은 예약어를 피해서 명확하게 작성한다.

</aside>
